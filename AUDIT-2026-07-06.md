# AgentFM Source Audit — 2026-07-06

Full-source audit of `agentfm-go`, `agentfm-desktop`, and `agentfm-python`. Every finding below was produced by a scoped auditor and then adversarially re-verified against the actual code (critical/high findings by two independent verifiers). **81 findings confirmed, 2 disputed, 7 refuted.** Line numbers are approximate (drift a few lines).

Severity shown as `reported → verified` where a verifier adjusted it.

---

## P0 — Ship blockers (build/release broken, or worst security)

### 1. `make` / `make build` is broken out of the box — `medium`
`agentfm-go/Makefile:11,97,100` — `build: build-agentfm build-relay` and `build-relay` compiles `RELAY_MAIN_FILE := ./cmd/relay`, but `cmd/relay/` contains **no `.go` files** (only a stale `build/` dir of prebuilt binaries). `go build ./cmd/relay` fails with "no Go files". The relay was folded into `agentfm -mode relay`; `scripts/release.sh` already builds only `./cmd/agentfm`. The Makefile still references the removed package in `build`, `build-all`, `install`, `download-install`.
**Fix:** delete `build-relay` and all `RELAY_*` vars/targets.

### 2. Windows desktop is DOA — backend binary can never be located — `high` (from completeness critic; verified)
`.github/workflows/desktop-windows-build.yml:34-35` builds `../agentfm-desktop/resources/agentfm-windows-amd64.exe` (Go `GOOS/GOARCH` names), but `agentfm-desktop/electron/backend-manager.ts:344-346` resolves `resources/bin/agentfm-${process.platform}-${process.arch}.exe` = `resources/bin/agentfm-win32-x64.exe` (Node names). **Two mismatches:** wrong directory (`resources/` vs `resources/bin/`) *and* wrong name. `build-binaries.sh`'s own comment warns this leaves Windows installs unable to find the backend. The shipped NSIS installer's app never starts its backend.
**Fix:** align CI output path/name with the resolver (emit `resources/bin/agentfm-win32-x64.exe`), or make the resolver map Node→Go arch names.

### 3. Desktop web typecheck fails — `tsc -p tsconfig.web.json` exits non-zero (8 errors / 4 files)
- `src/routes/Settings.tsx:134` — **runtime crash, not just types:** `window.api.app.saveSwarmKeyFile(...)` is not exposed by `electron/preload.ts` and has no `ipcMain` handler. Clicking **Export .key** throws `saveSwarmKeyFile is not a function`. (TS2339) — `high`
- `src/components/TopBar.tsx:40,56,62,71` — `WebkitAppRegion` not in React `CSSProperties` (TS2353/TS2559). — `high→low`
- `src/components/projects/ProjectDropdown.tsx:17` — `useEffect` cleanup returned only inside `if (open)`; `noImplicitReturns` → TS7030. — `high→low/medium`
- `src/routes/Assets.tsx:12` and `src/lib/ipc.ts` — `import type … from '../../electron/preload'` is outside `tsconfig.web`'s `include: ["src/**/*"]` → TS6307. — `high→low`
**Fix:** add the missing preload binding + IPC handler for `saveSwarmKeyFile`; augment `CSSProperties` (or cast); add `return undefined`; move shared IPC types into `src/`.
**Note:** `electron-vite build` does **not** typecheck, so nothing catches these at build time.

### 4. `ZipDirectory` follows symlinks out of the sandbox — worker host-file exfiltration — `high`
`agentfm-go/internal/utils/artifacts.go:66` — `filepath.Walk` (lstats) + `copyFileIntoZip` → `os.Open(path)` **follows symlinks**. A containerized agent (running a Boss-supplied prompt) can `ln -s ~/.ssh/id_rsa /tmp/output/x` inside the bind-mounted output dir; `os.Open` resolves it on the **host** and ships the worker's private keys / `swarm.key` back to the Boss in the artifact zip.
**Fix:** in the Walk callback, skip entries where `info.Mode()&os.ModeSymlink != 0`.

### 5. Unbounded worker-stdout reads → gateway OOM — `high`
- `agentfm-go/internal/boss/openai.go:456` (`drainTaskStream`) — `io.Copy(&buf, filtered)` with **no `io.LimitReader`**. `timeoutReader` only enforces an *idle* deadline (refreshed each Read), so a worker streaming continuously fills `buf` without bound. Violates CLAUDE.md §1.1 (LimitReader before decode).
- `agentfm-go/internal/boss/api_async.go:201` (`runAsyncTask`) — buffers all worker stdout into `outputBuffer` that is **never read** (webhook payload is only `{task_id, worker_id, status}`). Unbounded read into a discarded buffer.
**Fix:** wrap in `io.LimitReader` with a max cap (as the artifact handler does); drain the async path with `io.Copy(io.Discard, …)`.

## P1 — Serious correctness / security

### Ledger
- **Append is not atomic** — `internal/ledger/impl.go:265`. `l.mu` is dropped between `tree.Append`+`signNewHead` (phase 1) and `WriteHead`+`lastHead=` (phase 2, after `proto.Marshal`). Between them `Prove` (called from `api_reputation.go:320`) sees `tree.Size()==N` but `lastHead.TreeSize==N-1`, so inclusion proofs are built over an N-leaf tree anchored to a size-N−1 head and **fail verification**. Concurrent Appends (HTTP handlers, execute, hourly rater) can also commit `head_N` then `head_{N-1}`, regressing `lastHead`. `high→high/medium`. **Fix:** hold `l.mu` across the whole Append.
- **Inbox fork acceptance** — `internal/ledger/store/inbox.go:99` + `internal/ledger/inbox/inbox.go:223`. `InsertInboxEntry` unconditionally upserts the per-peer chain head even when the entry insert no-ops, and never checks `current head == prevHash` in-tx. A rater signs `A` + two children `B1,B2` both `prev=A`; if `B1,B2` arrive first (orphaned) then `A`, `acceptAndPromote` promotes **both** with no re-check — an equivocation is absorbed instead of detected. `high→high/low`. **Fix:** conditional head update (only when row inserted AND `last_hash == prevHash`); promote ≤1 child per parent and flag the rest as equivocation.
- **Crash between AppendEntry and WriteHead permanently desyncs tree/head** — `impl.go:364`. Restart rebuilds an N-leaf tree but loads the size-N−1 head; `Prove` emits unverifiable proofs until the next Append heals it, with no size cross-check. `medium`. **Fix:** verify `lastHead.TreeSize == tree.Size()` at Open/Prove and re-sign, or persist entry+head in one tx.
- **Fetch clients allow ~1 GiB alloc per page** — `internal/ledger/fetch.go:183` (and `inbox_fetch.go:183`). Only per-entry (1 MiB) and per-page-count (1000) caps; no aggregate byte budget. A hostile responder (up to 5 untrusted peers via `CatchUpInbox`) serves 1000×1 MiB, all retained. `medium→low`. **Fix:** aggregate size budget, Reset on exceed.
- **Six ledger stream endpoints `Close()` instead of `Reset()` on error** — `fetch.go:52,147`, `catchup.go:72,115`, `inbox_fetch.go:47,147`. CLAUDE.md §1.1 hard rule: error paths must `Reset()`; a FIN on a half-consumed exchange signals success and leaks connections. `medium→low`. (The `catchup.go` head-fetch pair was also independently flagged by the completeness critic.)
- **Inbox `AcceptOrQueue` check-then-act race** — `inbox/inbox.go:166`. Read chain head + insert are separate store calls; gossip subscriber and bootstrap catch-up run concurrently, so two same-`prev_hash` entries can both be accepted. `medium→low`. **Fix:** serialize per-rater or single-tx compare-and-extend.
- **Orphan table DoS** — `inbox/inbox.go:254`. Global cap, no per-peer quota, no TTL/eviction; orphans persist in SQLite across restarts. A sybil gossips 10k signed entries with random `prev_hash`; once the cap fills, every honest reordered entry is dropped forever. `medium`. **Fix:** per-peer quota + age-based eviction (`received_at` exists).

### Worker / network
- **Container not killed on task-stream reset** — `internal/worker/handler.go:156`. Watcher polls `s.Conn().IsClosed()`, but the boss↔worker connection also carries telemetry streams, so it stays open when the Boss `Reset()`s only the task stream (HTTP client hangup). The Podman sandbox then runs to the full `TaskExecutionTimeout` (~10 min), burning CPU/GPU and holding a concurrency slot. Violates the "stream death SIGKILLs the container instantly" rule. `high→high/medium`. **Fix:** watch stream death (blocking `s.Read` → cancel `taskCtx` on `ErrReset`), not connection closure.
- **Artifact zip written directly to final path** — `internal/network/artifacts.go:201`. `os.Create(destPath)` where `destPath = agentfm_artifacts/<taskID>.zip`; `api_async.go:57` polls `os.Stat(zipPath)` and fires the "completed" webhook the instant the 0-byte file exists — mid-download. On `io.Copy`/truncation error the partial zip is never removed, so later reads treat a corrupt zip as the artifact. `high→low/medium`. **Fix:** download to `<taskID>.zip.part`, `os.Rename` on success, `os.Remove` on failure.
- **`ZipDirectory` swallows the `zip.Writer` finalize error** — `internal/utils/artifacts.go:24`. `defer archive.Close()` discards the central-directory-write error (ENOSPC/I/O), so a truncated zip is returned as success and shipped to the Boss. `medium`. **Fix:** close explicitly after Walk and propagate.

### Boss HTTP / OpenAI
- **Telemetry PeerID spoofing / radar poisoning** — `internal/boss/boss.go:295`. `profile.PeerID` comes from attacker-controlled gossip JSON and is never compared to `msg.GetFrom()`; `handleTelemetryProfile` does `b.activeWorkers[profile.PeerID] = profile`, so any peer can inject or overwrite another peer's radar entry. `medium`. **Fix:** reject unless `profile.PeerID == msg.GetFrom()`.
- **CSRF: `Origin: null` trusted** — `internal/boss/api.go:88`. `if origin == "" || origin == "null" { return true }`. Meant for the Electron `file://` renderer, but sandboxed iframes and `data:` URLs on any malicious site also serialize `Origin: null`. In default token-less loopback mode this lets any web page drive state-changing POSTs (`/api/execute`) against `127.0.0.1`. `medium`. **Fix:** don't trust `null` for unsafe methods; gate `file://` behind a desktop flag or require the API token.
- **Async path uses the pterm-spinner dialer** — `internal/boss/api_async.go:166`. `runAsyncTask` (up to 256 concurrent goroutines) calls `b.dialOmni` → `pterm.DefaultSpinner`, whose `Start` does an unsynchronized append to a package-global slice. The sibling `handleExecuteTask` was deliberately switched to the spinner-free `dialWorkerStream` to avoid exactly this race; the async path was missed. `high→medium` (data race + garbled logs; crash possible but not typical). **Fix:** use `b.dialWorkerStream`.

### Python SDK
- **`tasks.run` always waits the full `artifact_timeout` on NO_FILES tasks** — `client.py:147` and `async_client.py:121`. `run()` consumes `stream()` (whose `SentinelFilter` parses `[AGENTFM: NO_FILES]`/`FILES_INCOMING`) but discards markers and then *unconditionally* polls for `<task_id>.zip`. The Go boss writes that zip only on `FILES_INCOMING`, so any no-artifact task with `artifacts_dir` set blocks the full 120s. `high→high` (sync) / `high→medium` (async). **Fix:** only call `collect_for_task` when an `artifacts_incoming` marker chunk was observed.
- **Non-idempotent POSTs retried on `ReadError`/`RemoteProtocolError`** — `_transport.py:240`. The `idempotent` flag gates only the *status-code* retry, not the *exception* retry. A `POST /api/execute/async` (or `/v1/chat/completions`) that fails while reading the response is re-issued — but those errors mean the request reached the server, and the async ack invariant commits the work before the 202. Double-submit. `medium`. **Fix:** for non-idempotent methods retry only errors proving the request never left (`ConnectError`/`PoolTimeout`/`WriteError`).

### Desktop renderer
- **Metrics polling never resumes after the window is hidden once** — `src/hooks/useMetricsPoll.ts:51`. On a hidden tick, `tick()` early-returns without clearing `timerRef.current`, so on return-to-visible `!timerRef.current` is false and polling never restarts until the Dashboard remounts. Contradicts the hook's own doc. `high→medium`. **Fix:** null `timerRef.current` before the hidden early-return.
- **`saveSessions` trims the wrong end — newest chats dropped** — `src/lib/sessions.ts:22`. Sessions are newest-first; `slice(-50)` keeps the **oldest** 50 and discards the newest once >50. `medium`. **Fix:** `slice(0, LIMIT)`.
- **API explorer stuck on "Sending…" forever** — `src/components/developer/EndpointDetail.tsx:100`. The endpoint-change reset bumps `reqToken` but never resets `loading`, and the invalidated in-flight request returns before `setLoading(false)`. Switch endpoints mid-request (e.g. `/api/execute`, which streams for the whole task) and the Send button is disabled permanently. `medium`. **Fix:** also `setLoading(false)` in the reset block.
- **Stale artifact-poll loop bleeds into the next dispatch** — `src/hooks/useDispatch.ts:59`. The 10s poll never checks `ctrl.signal`, so an old task's loop can set `hasArtifact:true` on a new dispatch; and `checkArtifact` isn't `try/catch`ed here, so one IPC rejection wipes a completed dispatch's output. `medium→low`. **Fix:** bail on abort/supersede; wrap in try/catch.

## Disputed (mechanism real, worst-case impact conditional)

- **EigenTrust weights negative raters by `math.Abs`** — `reputation/eigentrust.go:212`. `weight := math.Abs(raterScore)` gives a −1.0 peer the same voting weight as a +1.0 seed (canonical EigenTrust clamps `max(s,0)`), and `ingest()` applies no equivocator/ejection filter. **Confirmed as a real deviation.** The DoS escalation is disputed: the finding's "−0.5 default dispatch floor" is wrong — `reputationFloor` defaults to −1.0 (`boss.go:196`), so with default config the floor gate never rejects. Real smell, no default-config DoS. Worth fixing the weighting + wiring the floor.
- **useChat cross-project session save** — `src/hooks/useChat.ts:51`. On an `activeProjectId` change the save effect runs once with the previous render's stale `sessions`, doing `saveSessions(newId, oldSessions)`. **Self-corrects** when `loadSessions(newId)` resolves — *except* under a rapid A→B→C switch that cancels B's load, leaving B's storage permanently holding A's chats. Real transient bug; permanent-corruption path is an edge case.
